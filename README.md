# Apartment_Price_Visor

Сервис для автоматического сбора и анализа объявлений о продаже квартир с целью последующей оценки справедливости цены.

## Что уже реализовано

На текущем этапе в проекте собрана базовая инфраструктура для воспроизводимого data pipeline и автоматического пополнения сырого датасета.

### 1. Инициализация и структура проекта

* Инициализирован Python-проект с использованием `uv`.
* Настроена базовая структура директорий в стиле ML/data-проекта:

  * `src/` — исходный код,
  * `data/` — сырые и обработанные данные,
  * `tests/` — тесты,
  * `configs/` — конфигурации,
  * `reports/` — артефакты и отчёты.
* Настроен `.gitignore`.
* Инициализирован Git-репозиторий.

### 2. Настройка DVC

* Подключён `DVC` для отслеживания тяжёлых артефактов.
* Настроено хранение изображений объявлений через DVC вместо Git.
* В репозиторий добавлены служебные файлы DVC (`config`, `.dvc`-метафайлы), а сами изображения выведены из обычного Git-tracking.

### 3. Реализован парсер карточки объявления Move.ru

Разработан парсер отдельного объявления с Move.ru, который извлекает:

* ссылку и `listing_id`,
* цену и цену за м²,
* дату добавления и просмотры,
* количество комнат,
* общую, жилую площадь и площадь кухни,
* этаж и этажность,
* тип жилья и тип объекта,
* характеристики ремонта и объекта,
* адресные признаки,
* ближайшее метро,
* сведения о ЖК,
* историю изменения цены,
* текстовые поля (`title`, `description`),
* список URL изображений.

### 4. Реализован поиск объявлений в поисковой выдаче Move.ru

Сделан отдельный модуль, который:

* проходит по страницам поисковой выдачи Move.ru,
* извлекает URL карточек объявлений,
* получает `listing_id`,
* поддерживает дедупликацию,
* позволяет в будущем использовать инкрементальное обновление датасета через поиск только новых объявлений.

### 5. Реализован загрузчик изображений

Сделан отдельный модуль загрузки изображений по `image_urls`, который:

* скачивает изображения объявления,
* сохраняет их в структуру вида `data/raw/move_ru/images/<listing_id>/...`,
* возвращает локальные пути к сохранённым изображениям.

### 6. Собран первичный raw-pipeline

Реализован orchestration pipeline, который объединяет все шаги в единый процесс:

1. поиск новых URL объявлений,
2. парсинг карточек,
3. объединение результатов в сырой JSONL-датасет,
4. скачивание изображений,
5. подготовка изображений к сохранению через DVC.

На выходе формируются:

* `data/raw/move_ru/listings.jsonl` — сырой датасет объявлений,
* `data/raw/move_ru/errors.jsonl` — лог ошибок парсинга,
* `data/raw/move_ru/images/` — изображения объявлений.

## Текущий статус проекта

Сейчас проект находится на стадии формирования **raw data layer**:

* инфраструктура проекта настроена,
* сбор данных с Move.ru реализован,
* пайплайн пополнения сырого датасета подготовлен,
* хранение изображений вынесено в DVC.

## Следующие шаги

* построить модуль `preprocessing/build_dataset.py` для преобразования `listings.jsonl` в чистый табличный датасет;
* добавить очистку данных, удаление дублей и приведение типов;
* построить baseline-признаки для задачи оценки справедливой цены;
* подготовить baseline-модель для предсказания рыночной стоимости объявления;
* реализовать регулярное автоматическое пополнение датасета новыми объявлениями.

## Используемый стек

* Python
* uv
* requests

## Идея проекта

Цель проекта — построить систему, которая сможет:

* автоматически собирать объявления о продаже квартир,
* формировать обновляемый датасет по рынку недвижимости,
* анализировать характеристики объектов,
* оценивать, насколько цена объявления соответствует ожидаемой рыночной стоимости.
